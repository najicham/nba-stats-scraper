# NBA Grading Audit — 2026-05-18

**Trigger:** Critical reviewer's meta-question from `2026-05-18-3` handoff. The MLB grading audit (5/18 session) revealed reported MLB BB HR was 58.33% under a void rule books don't honor and 53.85% under what bettors actually face — a 4.48pp inflation. Run the same audit on NBA before more model/filter decisions get made against the reported 63.8% NBA BB HR.

## Verdict

**NBA grading is aligned with book practice. Reported BB HR is NOT inflated by hidden quality gates.** No analog to the MLB MIN_IP issue.

## Method

1. Enumerate void conditions in `data_processors/grading/prediction_accuracy/prediction_accuracy_processor.py:detect_dnp_voiding`.
2. Compare each to common-book practice (DK/FD NBA player props grade "1 second of game action" = bet stands; void only if player did not play at all).
3. Quantify each void category from `prediction_accuracy` for the 2025-10-01 → 2026-04-15 window.
4. Verify the BB-HR formula in `ml/analysis/league_macro.py` excludes voids from the denominator (it does).

## Void inventory (whole-season, 2025-26)

```sql
SELECT is_voided, void_reason, COUNT(*) AS n,
       COUNTIF(actual_points > 0) AS played_n,
       COUNTIF(minutes_played > 0) AS min_n
FROM `nba-props-platform.nba_predictions.prediction_accuracy`
WHERE game_date >= "2025-10-01" AND game_date <= "2026-04-15"
  AND has_prop_line = TRUE AND recommendation IN ("OVER","UNDER")
GROUP BY 1, 2 ORDER BY n DESC
```

| `is_voided` | `void_reason` | `n` | `played_n` | `min_n` | book aligned? |
|---|---|---|---|---|---|
| true | `dnp_unknown` | 1,088 | 0 | 0 | yes — 0 min, 0 pts → book also voids |
| true | `dnp_injury_confirmed` | 459 | 0 | 0 | yes |
| true | `dnp_late_scratch` | 288 | 0 | 0 | yes |
| false | NULL (graded WIN) | 17,405 | 16,823 | 17,400 | n/a (graded) |
| false | NULL (graded LOSS) | 14,892 | 14,501 | 14,889 | n/a (graded) |
| false | NULL (`prediction_correct IS NULL`) | 58 | 58 | 58 | **push** (actual = line); book also refunds |

**No void category has `played_n > 0` or `min_n > 0`.** No NBA pick is voided when the player actually played any minutes. Compare to MLB where `min_ip < 4.0` voided pitchers that DK pays at `0.33 IP`.

## Code review — `detect_dnp_voiding`

```python
dnp_detected = (
    is_dnp or
    (actual_points == 0 and (minutes_played is None or minutes_played == 0)) or
    actual_points is None
)
```

Trigger conditions:
1. `is_dnp = TRUE` — `player_game_summary` flag (upstream).
2. `actual_points == 0 AND (minutes IS NULL OR == 0)` — zero stats line.
3. `actual_points IS NULL` — boxscore row missing.

All three correspond to "player did not play at all" in book terms. **There is no minutes threshold** that books would honor differently.

## Push handling

58 rows with `is_voided=false, prediction_correct=NULL, actual_points = line_value`. These are pushes. Book treatment: refund. Grader treatment: NULL → excluded from `bb_hr_7d` denominator (`league_macro.py:253-255`: `COUNTIF(prediction_correct IS NOT NULL)`). Consistent.

## BB-level verification

Joining `signal_best_bets_picks` to `prediction_accuracy` on `(game_date, player_lookup, recommendation, line_value)` for 2025-10-01 → 2026-04-15:

| outcome | n |
|---|---|
| WIN (`prediction_correct=TRUE`) | 415 |
| LOSS (`prediction_correct=FALSE`) | 235 |
| void (`dnp_late_scratch`) | 23 |
| void (`dnp_unknown`) | 14 |
| void (`dnp_injury_confirmed`) | 7 |

→ 415 / (415 + 235) = **63.85%** — matches the memory's "Season record: 415-235 (63.8%)".

All 44 BB voids have `minutes_played = 0/NULL` and `actual_points = 0/NULL`. Every void would also be voided by DK/FD. No false voids inflating the HR.

(Note: the 650 BB-graded count is system-replicated — joining without `system_id` produces ~3.2× the distinct BB pick count. Both numerator and denominator scale uniformly, so HR is invariant. The replicated count is what historical reports have used; treat it as "BB-pick-system rows" rather than "distinct BB picks" if reporting needs to distinguish.)

## What is NOT in scope (deferred)

- **Push line resolution.** Some pushes (58 in 2025-26) may correspond to lines that books actually offered as half-points elsewhere. If a book offered `7.5` and we stored `7`, we'd push when bettors UNDER-win. Not a grading bug per se, but worth a line-source audit if push count grows.
- **System-replication count.** Reporting unit (`distinct BB picks` vs `BB pick × system`) is ambiguous in current dashboards. Cosmetic — doesn't affect HR.
- **Cross-book grading divergence.** This audit benchmarks against the standard DK/FD rule "1 second of action = bet stands". Books with stricter rules (rare for player points) would void some currently-graded picks; bettors at those books would face a lower HR.
