### Step 1: Create `v1/mlb/best-bets/all.json`

The frontend consumes `best-bets/all.json` (not per-date files). It needs:
```json
{
  "date": "2026-03-27",
  "season": "2026",
  "generated_at": "...",
  "record": {
    "season": { "wins": 0, "losses": 0, "pct": 0.0 },
    "month":  { "wins": 0, "losses": 0, "pct": 0.0 },
    "week":   { "wins": 0, "losses": 0, "pct": 0.0 },
    "last_10": { "wins": 0, "losses": 0, "pct": 0.0 }
  },
  "streak": { "type": "win", "count": 0 },
  "best_streak": { "type": "win", "count": 0 },
  "today": [ ...picks... ],
  "weeks": [ ...weekly history... ],
  "total_picks": 0,
  "graded": 0
}
```

**Option A (recommended): Extend `mlb_best_bets_exporter.py`**
- Add a second export method `export_all()` that:
  1. Queries BQ for today's picks → `today[]`
  2. Queries BQ for last 8 weeks of graded picks → builds `weeks[]`
  3. Computes `record` windows (season/month/week/last_10) from graded results
  4. Writes `v1/mlb/best-bets/all.json`
- Runs after the per-date export

**Pick shape in `today[]`** (mapped to NBA `BestBetsPick` shape):
```json
{
  "rank": 1,
  "player": "Gerrit Cole",
  "player_lookup": "gerritcole",
  "team": "NYY",
  "opponent": "BOS",
  "home": true,
  "direction": "OVER",
  "stat": "K",

  "edge": 2.0,
  "angles": [],
  "game_time": "2026-03-27T18:05:00Z",
  "actual": null,
  "result": null
}
```

Note: `stat: "K"` is the only MLB-specific field the frontend needs to handle.

### Step 3: Create Cloud Scheduler job

MLB games start ~7 PM ET. Picks should be exported by ~noon ET (before lineup locks):

```bash
# Morning: publish today's picks (run after predictions are generated ~9 AM ET)
gcloud scheduler jobs create pubsub mlb-daily-best-bets-publish \
  --project=nba-props-platform \
  --location=us-central1 \
  --schedule="0 14 * * *" \

  --message-body='{"sport":"mlb","export_types":["best-bets"],"target_date":"today"}' \
```

## BQ Tables Used by Exporters

```sql
-- Today's picks
SELECT pitcher_lookup, pitcher_name, team_abbr, opponent_team_abbr, is_home,
       strikeouts_line, predicted_strikeouts, recommendation, confidence, edge,
       model_version, is_correct, actual_strikeouts
FROM mlb_predictions.pitcher_strikeouts
WHERE game_date = @game_date
  AND recommendation IN ('OVER', 'UNDER')
  AND confidence >= 0.70

-- Season record window
SELECT COUNT(*) as total,
       COUNTIF(is_correct = TRUE) as wins,
       COUNTIF(is_correct = FALSE) as losses
FROM mlb_predictions.pitcher_strikeouts
WHERE game_date >= @season_start AND is_correct IS NOT NULL
```
