The scanner (modeled after NBA's `scripts/nba/training/discovery/feature_scanner.py`) will:

1. Load all `under_shadow` + `under_shadow_backfill` picks from `blacklist_shadow_picks`
2. Join with `prediction_accuracy` for ground-truth outcomes
3. For each candidate ranking feature (edge, signal_count, real_signal_count, each individual signal confidence, archetype tags), compute:
   - Spearman correlation with `prediction_correct`
   - Slate-relative ranking HR (within each game_date, does picking the top-N by this feature beat random?)
   - Cross-season consistency (2024 vs 2025 vs 2026 shadow)
   - Bonferroni-corrected significance test
4. Output ranked list of features that improve HR when used as ranking signal, with weights derived from regression coefficients on hit/miss

### Proposed production ranker

```python
# Generated from discovery — DO NOT hand-edit
UNDER_RANK_FEATURES = {
    # feature: weight (derived from regression coefficient)
    ...
}

def under_rank_score(pick):
    score = 0.0
    for feat, weight in UNDER_RANK_FEATURES.items():
        score += pick.get(feat, 0.0) * weight
    return score
```

## Sanity checks before adopting the ranker

Don't blindly ship whatever the scanner produces. Before promoting:

- **Cross-validation:** Hold out 20% of shadow data, train ranker on 80%, verify it improves top-5 HR on held-out 20% by at least 3pp vs random ranking
- **Sample size:** No feature in `UNDER_RANK_FEATURES` should be backed by fewer than 30 graded picks where it fired
- **Multiple-comparisons:** Apply BH-FDR correction across the feature scan. Memory note: NBA's discovery scanner does this in `stats_utils.py`
- **Archetype risk:** If the ranker top-loads picks from one archetype (e.g. 80% are elite-K-pitcher × low-line), that archetype better be the one with the highest HR in `02-AGENT-FINDINGS.md` Agent 5's analysis. If not, the ranker has overfit to a single feature.
