# Daily Orchestration Tracking

**Purpose:** Track issues, patterns, and improvements discovered during daily orchestration monitoring.

---

## Structure

| File | Purpose |
|------|---------|
| `ISSUES-LOG.md` | Log of issues found during daily checks |
| `VALIDATION-IMPROVEMENTS.md` | New queries/checks discovered that should be added |
| `PATTERNS.md` | Recurring patterns and their solutions |

---

## When to Update

### Log an Issue (`ISSUES-LOG.md`)
- When you find a data gap or failure during daily checks
- When a scraper fails unexpectedly
- When predictions are missing or incomplete

### Add a Validation Improvement (`VALIDATION-IMPROVEMENTS.md`)
- When you discover a new useful query
- When you find a better way to check something
- When you identify a gap in current validation

### Document a Pattern (`PATTERNS.md`)
- When you see the same issue 2+ times
- When you discover root cause of recurring problems
- When you find a reliable fix for a common issue
